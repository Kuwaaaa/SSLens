Lumen Privacy Policy

Last updated: April 27, 2026. This policy is written for the invite-only Lumen v2 beta.

Short Version

Lumen stores the Lens cards you create, the page they belong to, your handle, reactions, reports, and basic room presence needed to make the extension work. Anonymous Lens are anonymous to other users, not to the server operator.

What We Collect

  • Your invite redemption data: handle, user id, invite code usage, and bearer token metadata.
  • Lens data: body text, type, tags, references, anchor quote/position data, URL, room id, author id, anonymous flag, and creation time.
  • Reaction and report data: which user reacted to or reported which Lens, plus timestamps.
  • Presence data while a tab is connected by WebSocket, such as the room id and user id needed to show who is currently there.

URLs And Rooms

When the extension runs on an allowed page, it canonicalizes the page URL and derives a room id from it. Lens creation sends the canonical URL and room id to the server so other invited users on the same page can see the same Lens layer.

Anonymous Lens

If you post with the anonymous toggle on, other users see the author as Anonymous. The server still records your real user id for moderation and abuse handling. This is not zero-knowledge or operator-blind anonymity.

Reports

When you report a Lens, the server stores the reported Lens id, your user id, a reason value, and the time of the report. Reports are used only to help the operator review beta content.

Who Can See The Data

Other invited users can see non-hidden Lens content, public handles, anonymous display names, tags, references, and reactions in the rooms they visit. The server operator can access stored records for development, debugging, moderation, and backup.

What We Do Not Do

  • We do not sell beta data.
  • We do not use visible AI-authored Lens as fake user content.
  • We do not run a public signup flow in this beta.
  • We do not claim anonymous Lens are unlinkable from the operator.

Retention

Lens are designed to persist on a page during the beta. Operational records may be kept as long as needed to run the beta, debug the system, moderate content, and evaluate whether Lumen should continue.

Questions Or Removal

If you are in the invite-only beta and want a Lens removed or have a privacy question, contact the project operator through the beta group channel.
